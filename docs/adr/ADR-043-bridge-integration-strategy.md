# ADR-043 — Flutter-Rust Bridge Integration Strategy
> **Status:** Draft | 2026-03-29

## Context
With the pivot from `egui` to **Flutter**, we require a stable, high-performance bridge between our existing Rust game logic and the new Dart UI layer. We have chosen **flutter_rust_bridge (FRB) v2.0** as the primary integration mechanism.

## Decision: Bridge Architecture (Option A)

### 1. Sovereign Rust Logic
The original `OperatorGame` repository remains the single source of truth for all game mechanics, models, and persistence. No Flutter-specific code will be introduced into the original `src/` directory.

### 2. Bridge Crate Topology
We will establish a dedicated `native/` directory within `operator_game_flutter` that acts as the entry point for FRB.

```text
/operator_game_flutter
  /native
    /api.rs            <-- The bridge boundary (exports Rust functions to Dart)
    /Cargo.toml        <-- References the original OperatorGame crate
    /src/lib.rs        <-- FRB boilerplate linking api.rs
```

### 3. Data Transfer Strategy
- **State Push**: Flutter will poll the Rust `GameState` via a thin wrapper or use a `Stream` (FRB `Sink`) to receive real-time updates.
- **Commands**: UI interactions (e.g., `stage_operator`) will call asynchronous Dart functions generated by FRB, which execute the corresponding Rust logic.
- **Serialization**: FRB v2 will handle the automated conversion of Rust `struct` and `enum` types into Dart classes, including complex genetics models.

## Rationale
Using a separate bridge crate (Option A) ensures that the existing game logic can still be tested and built independently (e.g., `cargo test` in the original repo still works). It prevents "namespace pollution" and makes the Flutter shell truly a consumer of the Rust service.

## Consequences
- **Developer Workflow**: `flutter_rust_bridge_codegen generate` must be run whenever the `api.rs` boundary changes.
- **Build Pipeline**: Compilation will now involve both `flutter build` and `rustc` (via FRB's build scripts).
- **Tooling**: Requires `flutter_rust_bridge_codegen` (v2.0.0-dev.x) installed on the development host.
